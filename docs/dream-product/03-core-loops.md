# 03 — Core Loops

## 2 つの中核ループ + Compound moment

本アプリの体験は **3 つの動的要素** で構成される:

```
┌─────────────────────────────────────────────────────────────┐
│ Loop 1: 秘書ループ (Outsource Thinking)                      │
│ 「貯める → 整理 → 聞いたら答える」                            │
└─────────────────────────────────────────────────────────────┘
                          ↕ Compound moment
┌─────────────────────────────────────────────────────────────┐
│ Loop 2: 家庭教師ループ (Understanding)                       │
│ 「surface → 興味 → 深堀り → 腹落ち」                          │
└─────────────────────────────────────────────────────────────┘
                          ↕ Compound moment
┌─────────────────────────────────────────────────────────────┐
│ ★ Compound moment (両ループを繋ぐ瞬間)                       │
│ 「秘書 chat の答え / 家庭教師の深堀り答え が wiki に file     │
│  され、次の surface / 次の答え が一段リッチになる」          │
└─────────────────────────────────────────────────────────────┘
```

両ループは **独立して動く** が、**compound moment が両者を強化し合う flywheel** を作る。

---

## Loop 1: 秘書ループ (Outsource Thinking)

### 役割

ユーザーの「貯めたい」「あれ何だっけ」「これまとめて」を AI が代行する。
人間は **「読む / 整理する」労力から解放** される。

### 5 ステップ

```
1. 共有 (Share)
   ユーザーが iPhone Share Sheet 経由で情報を投入
   (Web 記事 / PDF / 写真 / スクショ / AI 会話)
   ↓
2. 抽出 (Extract)
   AI が読み解いて要点 / 概念 / 関係性を抽出
   完全 on-device、ユーザーは何もしない
   ↓
3. 蓄積 (Compound)
   既存の知識ベースに統合、関連する概念ページを更新
   1 ソース投入 → 10-15 ノードに波及更新
   ↓
4. 提示 (Surface)
   「News Clip 風カード」で「今のあなたへ」要点を提示 (受動)
   または 「秘書 chat」 で質問に答える (能動)
   ↓
5. 引用 (Cite)
   答えには必ず元記事 ID と引用が付く (ソース追跡)
   信頼性 + 検証可能性
```

### 入口 / 出口

| 入口 | 出口 |
|---|---|
| Share Sheet タップ | News Clip 風カード (受動) |
| Photo Picker | 秘書 chat 答え (能動) |
| 既存 reading list 一括 import (V2) | Widget glance (ambient) |

### 価値の本質

- **「貯めっぱなし」から脱却** (タブ太郎さん救済)
- **「あれ何だっけ」を 1 秒で解決** (全ペルソナ共通)
- **読まなくても要点が入る** (受動的キャッチアップ)

### このループだけだと足りないこと

- 「分かった気」になって終わるリスク
- 知識が「他人事」のまま、自分の中に定着しない
- 「ChatGPT に context が付いただけ」と感じられる可能性

→ **家庭教師ループ (Loop 2) で補う**。

---

## Loop 2: 家庭教師ループ (Understanding)

### 役割

ユーザーが **「自分のものにしたい」と思った瞬間に伴走** する。
人間が能動的に学ぶプロセスを、AI が壁打ち相手として加速する。

### 5 ステップ

```
1. Surface (今のあなたへ)
   起動 default = 学習タブ
   カード 1 枚 (概念 / 仮説 / 気付き) が surface される
   ユーザーは見るだけ、ストレスなし
   ↓
2. 興味判定 (✓ or 🤔)
   「✓ わかった」 → 次のカードへ
   「🤔 もっと」 → 深堀り chat に展開
   どちらも合言葉、不正解という概念がない
   ↓
3. 深堀り会話 (Chat)
   そのカードを context にして AI と対話
   「これって何で?」「実例ある?」「他とどう違う?」
   AI は wiki から関連を引いて answer
   ↓
4. 腹落ち判定 (User の主観)
   「自分の言葉で説明できる」感覚が来たら「✓ わかった」
   まだなら深堀り継続
   タイムアウトなし、強制終了なし
   ↓
5. ファイリング (Compound moment)
   会話で得られた insight が概念ページに自動 file
   次に同じ概念が surface されると、より richer なカードになる
```

### 入口 / 出口

| 入口 | 出口 |
|---|---|
| 学習タブ (default 起動) | 深堀り chat (能動展開) |
| Widget タップ | 「✓ わかった」スコア up |
| カード関連 link (連鎖) | 概念ページ更新 (compound) |

### 価値の本質

- **「読んだ気」から「腹落ち」へ** (Karpathy の "understanding")
- **能動学習が無理なく続く** (Calm UX、不安喚起ゼロ)
- **自分の理解度が wiki に蓄積** (userUnderstanding スコア)
- **興味ベースの探索** = 強制学習じゃない、好奇心ドリブン

### このループだけだと足りないこと

- そもそも「surface する材料」がないと始まらない
- ユーザーが能動的に開く必要がある

→ **秘書ループ (Loop 1) で材料を供給**。

---

## ★ Compound Moment (両ループを繋ぐ瞬間)

### 定義

**「秘書 chat の答え / 家庭教師の深堀り会話 の最後で、新しい insight が wiki に file される瞬間」**。

詳細は `docs/concept-review/karpathy-llm-wiki/07-external-references.md` の「Compound Moment」セクション参照。

### 発火条件 4 つ (詳細)

#### 条件 1: 秘書 chat 答えに引用元 ≥ 2 件あった時

**何が起きるか**:
- ユーザーが秘書 chat で質問 → AI が複数の保存記事を引用して答える
- 引用元 2 件以上 ある = **複数のソースを横断した insight が生まれた瞬間**
- アプリ側で自動的に:
  - その質問 + 答え + 引用記事 ID を **SavedAnswer @Model** として永続化
  - 答えで触れた概念 (entity / topic) の **概念ページに 「Q&A セクション」として append**
  - chat session 履歴に消えずに wiki の永続資産になる

**ユーザー体験**:
- chat 中はバックグラウンドで silent (Calm UX)
- 答えの末尾に「📌 この答えを保存」ボタン表示 (optional 明示)
- 知識 Clip タブの「Saved Answers」セクションで後から閲覧可能
- 次に同じ概念に関する質問をすると、過去の答えが context に入る

**例**:
- 質問: 「Apple Intelligence と Foundation Models の関係は?」
- AI 答え: 「Apple Intelligence は OS レベル機能群、Foundation Models はその基盤 LLM SDK。記事 [A] / [B] / [C] より」
- → SavedAnswer 自動保存、Apple Intelligence 概念ページに「Q&A: ~の関係」append
- 翌日「Apple Intelligence と他社の比較は?」と聞くと、前回の答えが context に乗ってリッチに

#### 条件 2: 家庭教師深堀り会話の終わり

**何が起きるか**:
- ユーザーが学習カードから「🤔 もっと」で深堀り会話開始
- 数ターン会話して、ユーザーが「✓ わかった」をタップ
- 会話全体を Foundation Models で analyze → 新しい insight (元のカードに無かった理解) を抽出
- 抽出 insight を **概念ページの「横断的知見」セクションに append**

**ユーザー体験**:
- 会話終了時に soft toast: 「✨ 概念 [Apple Intelligence] に新しい insight が追加されました」
- 次に同じ概念のカードが surface される時、insight が含まれてリッチに
- 自分が深堀りした内容が wiki に蓄積する = **「学んだ証」が残る**

**例**:
- カード: 「Apple Intelligence (基本)」
- 深堀り chat: 「なんで Apple は OpenAI と提携した?」「他の LLM との連携は?」「Privacy の仕組みは?」
- → 概念ページに「OpenAI 提携の背景」「Privacy Cloud Compute との関係」が追記
- 翌週 同じカード が surface されると「あなたが先週理解した内容」も含めて表示

#### 条件 3: 「✓ わかった」タップ (Understanding feedback)

**何が起きるか**:
- ユーザーがカードを見て「✓ わかった」をタップ
- 該当概念ページの `userUnderstanding` スコアを +1
- 関連概念 (graph で 1-hop 近傍) の `userUnderstanding` も +0.3 (波及)
- カードキュー優先度ロジックに反映: 「わかった」概念は surface 頻度が下がる、わからない概念が優先

**ユーザー体験**:
- タップだけ、即次のカードへ (Calm UX、爽快感)
- 「✓ わかった」が増えると、surface されるカードがどんどん未理解領域に集中する
- 数週間使うと「自分の理解度マップ」が wiki に蓄積されている状態
- Settings で「私の理解度」可視化 (任意、デフォルト非表示)

**注意**:
- ❌ 「不正解」概念は作らない (Calm UX 原則)
- ❌ 「テスト」感を出さない、「自己申告」のみ
- ❌ 連続日数 / streak を計測しない
- ✅ 「わかった」「もっと知りたい」の 2 値、定量化は内部のみ

#### 条件 4: 新記事 ingest 完了時

**何が起きるか**:
- ユーザーが Share Sheet で新記事を保存
- 抽出パイプライン完了 (要約 / KeyFact / Entity / Graph triple)
- 抽出された entity / concept のうち、既存の概念ページがあるものを検出
- 該当概念ページを `isStale = true` でマーク
- 次回 BGTask または 該当カード/ページが開かれた時に **再合成** (summary + crossSourceInsights を新ソース含めて生成し直す)

**ユーザー体験**:
- バックグラウンド処理、ユーザーは何もしない (Calm UX)
- アプリ次回開く時に「+ 新しい情報が反映されました」と概念ページに soft badge (任意)
- 「あなたが追っている人物・モノ」セクションで「最近更新」順に並ぶ
- 1 記事投入 → 関連概念ページ 10-15 個に波及更新が走る = **「使うほど賢くなる」の本体**

**例**:
- 既存概念ページ: 「Apple Intelligence」(7 記事から構築済)
- 新記事 ingest: 「Apple Intelligence v2 発表」(8 記事目)
- → Apple Intelligence 概念ページ `isStale = true`、再合成
- 新 summary: 「v1 から v2 への進化点を含む統合説明」
- 新 crossSourceInsights: 「v1 と v2 の機能比較」「業界の反応」等が追加

### 効果

```
共有 1 件 → 関連 10-15 ノード更新 (Loop 1)
   ↓
次回 surface → より richer な「今のあなたへ」(Loop 2)
   ↓
深堀り会話 → 新 insight が file (Compound)
   ↓
次回 query → より精度の高い答え (Loop 1)
   ↓
... 永久に成長する flywheel
```

### ハルシネーション位置の管理

Compound moment で wiki に書き込まれるのは:
- ✅ 引用元が確実な insight
- ✅ ユーザーが「わかった」と認めた理解度の更新
- ❌ AI の推測 / 一般知識からの補強 (これは絶対 file しない)

→ 「材料はレビュー済の事実、解釈は user 主体」を貫く。

## 3 つの動的要素の関係 (Flywheel 図)

```
              [情報共有]
                 ↓
       Loop 1: 秘書ループ
       貯める → 整理 → 答える
                 ↓
            [概念蓄積]
                 ↓
    ★ Compound moment 発火
                 ↓
            [リッチな wiki]
                 ↓
       Loop 2: 家庭教師ループ
       surface → 興味 → 深堀り → 腹落ち
                 ↓
       [更に深い insight が wiki に]
                 ↓
       (Compound moment が更に発火)
                 ↓
       (Loop 1 で答えが更にリッチに)
                 ↓
              ...無限ループ
```

**「使うほど賢くなる」は嘘ではない、両ループ × Compound moment の構造で実現**。

---

## どちらのループから入っても OK な設計

ユーザーごとに「自然な入口」は違うので、強制しない:

| ユーザータイプ | 主に使うループ |
|---|---|
| タブ太郎 (情報過食型) | Loop 1 主、Loop 2 はたまに |
| 学さん (学習者) | Loop 2 主、Loop 1 で材料を貯める |
| 作る花子 (クリエイター) | Loop 1 主、 export で外に出す |
| 好奇さん (知識探求型) | Loop 2 主、寝る前に受動消費 |

→ **両ループは独立して価値を持つ**、片方しか使わない user も welcome。
→ ただし **両方使うと compound 効果が最大化する** (これがアプリの真価)。

---

## アンチパターン (避ける設計)

| アンチパターン | なぜダメ |
|---|---|
| 「両ループを毎日使え」と push 通知 | calm UX 原則違反、ストレス源 |
| Loop 2 で「不正解」「正解率」を表示 | テスト感 = 不安喚起、学習意欲削ぐ |
| Loop 1 で「未読 50 件」赤バッジ | 読まないことへの罪悪感を作る |
| Loop 連鎖を user に強制 (1 回目わかった → 2 回目要求) | spaced repetition 風だがやりすぎ |
| Compound moment を visible すぎる UI で見せる | 「育ってる感」は嬉しいが、過剰だとうるさい (soft toast 程度に) |
| Loop 1 答えに「次の問い」を 5 個並べる | runbook pattern は有用だが、3 個までが calm |

## 各ループの「成功 KPI」(内部議論用、ユーザーには見せない)

| KPI | Loop 1 | Loop 2 |
|---|---|---|
| Engagement | 1 セッションあたりの query 回数 | 1 セッションあたりのカード消費数 |
| Quality | 答えへの「✓ わかった」率 | 深堀り後の userUnderstanding 上昇率 |
| Compound | 1 query で更新される概念ページ数 | 1 深堀りで file される insight 数 |
| Retention | 月間ユーザーが Loop 1 を使った日数 | 月間ユーザーが Loop 2 を開いた日数 |

→ **KPI は UI に出さない** (calm UX)、内部分析と改善のため。
