# 09 — Naming Candidates

## このファイルの目的

製品名候補と評価軸を集約し、最終決定までの議論ログを残す。
仮称 "i knowledge base" を含む 20 候補を提示、評価軸で比較。

---

## 評価軸 (7 軸)

| 軸 | 中身 | 重要度 |
|---|---|---|
| **意味の伝達** | 名前から「何のアプリか」が一発で伝わるか | ★★★ |
| **覚えやすさ** | 短く / 発音しやすく / タイポしにくい | ★★★ |
| **既存被り** | 既存サービス / 商標と衝突しないか (Cortex 等は注意) | ★★★ |
| **ローマ字 / 日本語両対応** | 日本国内 + グローバル両方で通る | ★★ |
| **App Store SEO** | 検索でヒットしやすい (固有 + 一般語 mix) | ★★ |
| **ロゴ化** | ロゴデザインに落としやすい (短い / 象徴的) | ★ |
| **vision との整合** | 「思考代行 + 理解伴走」「あなた専用 AI」感が伝わるか | ★★★ |

## 候補 20 案

### Group A: 仮称 (user 提案)

#### 1. i knowledge base

- **意味**: 「あなたの知識ベース」、i が「Apple 風 + 個人 (I)」を兼ねる
- **強み**: 直球、機能が伝わる、Apple 製品っぽい
- **弱み**: 平易すぎる、商標として弱い、SEO が一般語との衝突大
- **評価**: ★★★☆☆ (継続検討、ただし最終ではない)

---

### Group B: 「知識」系日本語 (現知積系統)

#### 2. 知積 (chiseki) ★現アプリ名

- **意味**: 知識を積み重ねる、「知の蓄積」
- **強み**: 既存知名度 (もしあれば)、和語、固有性高
- **弱み**: 海外で読みにくい、新カテゴリ感が薄い
- **評価**: ★★☆☆☆ (継続するなら継承パターン、新ブランディングなら避ける)

#### 3. 知恵箱 (chiebako)

- **意味**: 知恵が貯まる箱
- **強み**: 親しみやすい、日本人にすぐ伝わる
- **弱み**: 海外展開に弱い、古風

#### 4. 知恵泉 (chiezen)

- **意味**: 知恵の泉、湧き出る
- **強み**: 詩的、湧き続けるイメージ
- **弱み**: 商標衝突可能性 (調査要)、海外発音難

---

### Group C: 英語 + 機能ベース

#### 5. KnowAtlas

- **意味**: Knowledge + Atlas (地図)、知識の地図
- **強み**: 「自分の興味の地図」イメージ強い、固有性高
- **弱み**: やや長い (9 文字)
- **評価**: ★★★★☆ (有力候補)

#### 6. Brainery

- **意味**: Brain + ery (場所)、頭脳のための場所
- **強み**: 親しみやすい、可愛い
- **弱み**: 学校 / 教育アプリと混同される可能性

#### 7. Compounder

- **意味**: Compound (複利) + er (主体)、複利で増やすもの
- **強み**: vision の「複利蓄積」直球、Karpathy 思想
- **弱み**: 金融用語と被る、堅い

#### 8. iWiki

- **意味**: 個人 (i) の wiki
- **強み**: 短い、Wikipedia 連想で機能が伝わる
- **弱み**: Apple "i" 接頭辞 商標訴訟リスク

#### 9. Memex (Mobile / Lite)

- **意味**: Vannevar Bush 1945 提唱の Memex
- **強み**: 知識マニアに刺さる、深い思想背景
- **弱み**: 一般人に伝わらない、ニッチ

#### 10. MindStack

- **意味**: 心の stack、思考の積み重ね
- **強み**: ITエンジニア層に刺さる、tech 感
- **弱み**: 一般人に「stack」が分かりづらい

#### 11. Cortex

- **意味**: 大脳皮質
- **強み**: 短い、強い、知性連想
- **弱み**: ⚠️ **既存被り大** (Snowflake Cortex / Palo Alto Cortex / airCloset cortex)、避けるべき

#### 12. Synapse

- **意味**: 神経シナプス、繋がり
- **強み**: 「繋ぐ」中核価値 と integral、科学的
- **弱み**: ⚠️ 既存被り可能性 (調査要、ノートアプリで使われてる可能性)

#### 13. Recall

- **意味**: 思い出す、呼び起こす
- **強み**: 短い、core value 直球
- **弱み**: ⚠️ 既存被り大 (Microsoft Recall、商標衝突)

#### 14. Tinker

- **意味**: いじくる、試す、組み立てる
- **強み**: 短い、遊び心、学習・探求 ニュアンス
- **弱み**: 知識アプリ感が薄い

## 「i knowledge base」をどう扱うか

仮称として使い続け、vision / 機能仕様 が固まった後の最終決定段階で:

**選択肢 A**: そのまま採用
- pros: ユーザー提案、シンプル、機能直球
- cons: 商標弱い、SEO 弱い、ロゴ化困難

**選択肢 B**: 別候補に切り替え (例: KnowAtlas)
- pros: 商標 / SEO / ブランディング有利
- cons: ユーザー提案を覆す形

**選択肢 C**: ハイブリッド (例: "KnowAtlas — Your i Knowledge Base")
- pros: 主称 + サブタイトルで両立
- cons: ややくどい

→ **dream product spec 完成後に user と最終決定**

## 名前を決める前にやるべきこと

1. **dream product spec 全体が固まる** (Phase 4 完了)
2. **migration plan を見る** (現知積からの距離が分かる)
3. **3-5 候補に絞る**
4. **商標 / ドメイン 調査** (上記)
5. **小数の friends で blind test** (どの名前が「Apple Intelligence をあなた専用に進化させる第二の脳」を連想させるか)
6. **最終決定 → ブランディング / ロゴ / app icon 制作**

→ V1 リリース直前まで「i knowledge base (仮)」のまま開発進行も可。
