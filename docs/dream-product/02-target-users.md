# 02 — Target Users

## ターゲット主軸

**一般の iPhone ユーザー (年齢 / 職業不問、より広く)**。
ただし「使う」と「ハマる」は別なので、4 つの代表的なペルソナで「響き方の幅」を可視化する。

共通項:
- iPhone を日常的に使う (Share Sheet が分かる程度)
- 普段 何かしらの情報 (記事 / SNS / 動画 / AI 会話) に触れている
- 「読みっぱなし」「貯めっぱなし」が気になっている
- プログラミング / Markdown / Obsidian の知識は **必要ない**

---

## ペルソナ 4 種

### ペルソナ 1: 情報過食型「タブ太郎さん」(35 歳、IT 系会社員)

**現状**:
- ブラウザに常時 80+ タブを開きっぱなし
- 「あとで読む」のために Pocket / Apple Reading List も使うが、ほぼ読み返さない
- 業務関連 + 趣味 + ニュース を雑食的に保存
- 「あの記事どこ行ったっけ」が頻発

**Pain**:
- 貯めるばかりで活用していない罪悪感
- 読んでいない事実が積み上がるストレス
- 「読み返さなきゃ」のプレッシャーで開けなくなる悪循環

**このアプリで救われるところ**:
- Share Sheet で 1 タップ保存、もう「あとで読む」プレッシャーから解放
- 読まなくても AI が要点を整理してくれる
- 「あの記事の要点だけ思い出したい」を秘書 chat で即答
- 通勤中に News Clip 風カードで「今のあなたへ」眠っている知識が surface
- ストレスゼロで「貯めっぱなし」が活きる

---

### ペルソナ 2: 学習者「学さん」(22 歳、大学生 / 23 歳、資格勉強中の社会人)

**現状**:
- 専門書 / 論文 / 講義動画 / オンライン記事 を日々消費
- ノート (Apple Notes / Notion / 紙) は取るが、検索・統合できない
- 「読んだはずなのに思い出せない」「概念が繋がらない」が頻発
- Anki などで暗記はするが、理解とは別

**Pain**:
- インプット量に対して定着率が低い
- ノートが分散して、横断できない
- 試験前に「あれどこに書いた?」で時間を浪費
- 学んだはずの概念が、別の文脈で出てきた時に繋がらない

**このアプリで救われるところ**:
- PDF / 写真 (スクショした論文) / Web 記事すべて 1 か所に統合
- AI が概念ページを自動生成 (人物・モノ・テーマ別)
- 「Apple Intelligence と Foundation Models の関係って?」を秘書 chat で即答
- 「腹落ちしたい時」は家庭教師カードで深堀り会話、自分の言葉で説明できるまで伴走
- 試験前に「concept ページを 1 周読む」が新しい復習方法に

---

### ペルソナ 3: クリエイター・ライター「作る花子さん」(40 歳、フリーライター / ブロガー / 研究者)

**現状**:
- 「書く」「作る」ために大量の input が必要
- 取材記事 / 論文 / 業界レポート / SNS の発言 を継続的に追う
- Notion / Obsidian / Scrivener / Bear など試したが結局統合できない
- 「あの人があの場で言っていた一言」「あの研究の主張」を後で引用したいが見つけられない

**Pain**:
- リサーチ作業の摩擦が大きい
- 引用元の追跡が手作業で時間泥棒
- 「読んだ」と「使えた」のギャップが大きい
- 別々の文脈で見た情報が「実は同じ話」に気づくのが遅い

**このアプリで救われるところ**:
- スクショ / 写真 / Web 記事 すべて 1 タップ統合
- AI が cross-source synthesis → 「Apple と OpenAI が同じ議論をしている」と自動検出
- 引用元が常に保存 (ソース追跡)
- 「あの主張、誰が言ってた?」を秘書 chat で即答
- 書く前の「リサーチ整理」工程が圧倒的に楽になる
- export 機能で素材を ChatGPT / Gemini に渡して別の AI で文章生成 (V1 から可)

---

### ペルソナ 4: 知識探求型「好奇さん」(50 歳、趣味で歴史 / 科学を深掘り)

**現状**:
- YouTube 動画 / Wikipedia / 専門書 / ポッドキャストを趣味で楽しむ
- ノートはあまり取らない、興味の赴くままに消費
- 「これとあれって関係あるよね」を気付くのが楽しい
- でも 「あの時の気付き、なんだっけ」が起きがち

**Pain**:
- 楽しんで読んだ・観た情報が断片化して定着しない
- 「あれとこれの繋がり」を自分の頭の中だけで管理しきれない
- 体系化したいが、自分でノート整理する気はない

**このアプリで救われるところ**:
- 興味のあるものを Share / 写真で 1 タップ保存
- AI が勝手に概念ページ + コミュニティを作ってくれる
- グラフ表示で「興味の地図」が広がる感覚
- 寝る前に「今日育った概念」カードで眺める = 受動的に学ぶ楽しさ
- 自分の興味の体系が「自分専用 AI」として育つのが嬉しい

---

### ペルソナ 5: 子育て中の親「育子さん」(33 歳、フルタイム共働き、子供 2 人)

**現状**:
- 子育て / 教育 / 健康情報を Instagram / YouTube / ニュースで日常的に消費
- 「これは後で読みたい」と Bookmark するが、保育園送迎や家事で時間が消える
- 同じ悩みでも「あの記事どこだっけ」を探す時間がない
- ママ友との会話で「あの記事に書いてあったよ」と言いたいが、出典を覚えていない

**Pain**:
- 時間が細切れ、まとまった読書時間が確保できない
- 子供に関する情報は鮮度と信頼性両方欲しいが、探す余裕がない
- 「読まなきゃ」というプレッシャーが追加の負担になる

**このアプリで救われるところ**:
- Share Sheet で 1 タップ保存、読む時間はなくても貯まる
- 子供寝かしつけ後の 5-10 分でカードを 2-3 枚消費
- 「離乳食のタンパク質の目安は?」を秘書 chat で即答
- 「読まなきゃ」プレッシャーがゼロ (Calm UX)、隙間時間に活きる

---

### ペルソナ 6: シニア「七六さん」(67 歳、退職後、健康 / 歴史 / 旅行に興味)

**現状**:
- 朝の新聞 / TV ニュースに加えて、スマホで健康・年金・歴史記事を読む
- 字が小さい、操作が複雑だとすぐ諦める
- 家族 (息子・娘・孫) に「あの記事で読んだんだけど…」と共有したい
- でも記事のリンクを送るだけだと相手も読まない

**Pain**:
- 新しいアプリを覚えるのが億劫、設定画面に入りたくない
- 文字が小さい / タップ精度が悪い / 機能多すぎ で挫折経験あり
- 読んだ内容を家族と共有したいが、要点だけ伝える手段がない

**このアプリで救われるところ**:
- Share Sheet タップだけで動く、設定なし
- News Clip 風カードで「要点」だけ受動的に読める = ニュースアプリと同じ感覚
- 「腹落ちしたい時」だけ深堀り、普段は受動でOK
- export 機能で要点をメッセージにコピペして家族に共有
- 「Apple Intelligence をあなた専用に進化」という売りは シニア にも刺さる (大企業 Apple の信頼)

---

### ペルソナ 7: 経営者 / リーダー「営みさん」(45 歳、中小企業経営 or 部門長)

**現状**:
- 業界トレンド / 競合動向 / 法改正 / マネジメント書籍 を継続キャッチアップが必要
- 移動 + 会議で時間が取られ、まとまった input 時間がない
- ニュース app / X / LinkedIn / メルマガ で情報を流し見、保存しても読み返さない
- 部下にも「あの記事読んだ?」と共有したいが、リンクだけ送っても読まれない

**Pain**:
- 業界の重要動向を見落とすリスク (致命的)
- 判断材料となる情報を整理して比較したいが、時間がない
- 部下・社内に知識共有する手段が弱い (リンク共有では伝わらない)
- 「読んだ気になっていた」で意思決定の質が落ちる懸念

**このアプリで救われるところ**:
- 移動中に Share Sheet で 1 タップ保存、後で AI が整理
- 早朝 5 分 Widget で「今日のあなたへ」要点チェック
- 「先月の AI 業界の主要 trend は?」を秘書 chat で 要点 + 引用付きで即答
- 概念ページが時間とともに育つので、競合動向 / 法改正 trend を時系列で追える
- export 機能で要点を社内 Slack / メールに共有 (リンクだけより届く)
- 「腹落ち」が必要な重要判断時のみ深堀り

---

## 共通項 (7 ペルソナを貫く本質)

| 共通の悩み | 共通の解決 |
|---|---|
| 「貯めっぱなし」のストレス | 1 タップ保存 + AI 自動整理 |
| 「あれ何だっけ」の頻発 | 秘書 chat で即答 |
| 「読んだのに繋がらない」 | 概念ページの cross-source synthesis |
| 「整理する時間がない」 | bookkeeping は LLM が全部やる |
| 「自分のものになっていない」 | 家庭教師カード + 深堀り会話で腹落ち |
| 「他の AI とこのアプリどう違う?」 | あなた個人の context を持つ Apple Intelligence の進化版 |

## 利用シナリオ (時間軸別)

シナリオを 4 つの時間帯で見ると、本アプリが日常のどこに居るかが見える。

### シナリオ A: 朝の通勤 (8:00, 15 分)

**状況**: 電車で立っている、片手スマホ、雑にコンテンツ消費したい

**ユーザー行動**:
- アプリを開く → 起動 default が「学習」タブ (家庭教師カード一覧)
- 今朝の「今のあなたへ」カードが 1 枚出る (例: 「Apple Intelligence の進化」)
- ぼんやり読んで「✓ わかった」 or 「🤔 もっと」
- 「もっと」の場合は 2-3 ターン会話、納得したら「✓」
- 5 分で 3 枚消費して終わり

**価値**: 通勤時間が「ゆるい学習」になる、生活との接点が自然

---

### シナリオ B: 昼休み (12:30, 5 分)

**状況**: 業務で「あれ何だっけ」が発生、すぐ答えが欲しい

**ユーザー行動**:
- アプリを開く → AI チャットタブ
- 「先週読んだ Foundation Models の context window って何 token だっけ」と質問
- 秘書 chat が保存記事から該当箇所を引用付きで答える
- 必要なら原文記事に jump

**価値**: 自分が読んだものに即アクセスできる秘書、ChatGPT より context 良い

---

### シナリオ C: 業務中 / 学習中 (15:00, 30 分)

**状況**: 集中して何かを調べたい、深く理解したい

**ユーザー行動**:
- 関連記事を 3-5 本 Share Sheet で連続投入
- 自動で概念ページが更新される (バックグラウンド)
- アプリで「概念」タブを開いて、新しく育った人物・モノページを読む
- 不明点を家庭教師 chat で深堀り、納得したら「✓ わかった」

**価値**: 「リサーチ + 学習」が一連のフローで回せる、別アプリへの context 切り替え不要

---

### シナリオ D: 寝る前 (23:00, 10 分)

**状況**: 眠い、能動的に読みたくない、でも知的好奇心は満たしたい

**ユーザー行動**:
- アプリを開く → 学習タブ
- 「今日育った概念」カードが順に出る (受動的)
- ふと興味あるものに「🤔 もっと」、軽く chat
- 眠くなったら閉じる、罪悪感ゼロ

**価値**: 「Netflix 代わり」の知的時間、connect しなくても OK な気軽さ

## ペルソナと利用シナリオのクロス

| シナリオ \ ペルソナ | タブ太郎 | 学さん | 作る花子 | 好奇さん | 育子さん | 七六さん | 営みさん |
|---|---|---|---|---|---|---|---|
| A: 通勤 (受動) | ◎ メイン | △ | ○ | ◎ メイン | △ (家事中) | ○ | ◎ メイン |
| B: 昼休み (秘書 query) | ◎ メイン | ◎ | ◎ | ○ | ○ | △ | ◎ メイン |
| C: 集中学習 (能動) | ○ | ◎ メイン | ◎ メイン | ○ | △ | ○ | △ (週末のみ) |
| D: 寝る前 (受動) | ◎ | ○ | △ | ◎ メイン | ◎ メイン (子供寝かしつけ後) | ◎ メイン | △ |
| E: Widget glance (5 秒) | ○ | △ | ○ | ○ | ◎ メイン | ◎ メイン | ◎ メイン |

→ **7 ペルソナ全員が複数シナリオで価値を得る**、ペルソナ依存の機能はない。
→ Widget は **時間がない人 (育子 / 七六 / 営み)** にとって特に価値が大きい (1 タップで「今日の要点」を確認)。

## ターゲット拡張軸 (V2+ で検討)

V1 では一般 iPhone ユーザー (上記 4 ペルソナ) を主軸にするが、V2+ で広がる可能性:

| 拡張軸 | 中身 | V?  |
|---|---|---|
| iPad / Mac native アプリ | より広い時間帯 (デスクワーク) で使える | V3+ |
| 多言語 UI (英 / 中 / 韓 / 仏...) | グローバル展開 | V3+ |
| シニア向け UI 最適化 | 文字大きく、操作シンプル化 | V2 検討 |
| 子供 / 学生向け 安全モード | 不適切コンテンツフィルタ | V3+ |
| プロフェッショナル深掘り (医療 / 法律 / 学術) | ドメイン特化テンプレ | V3+ |

## 取らないターゲット

明示的に「このアプリは向かない」と決めておく:

| ターゲット | 理由 |
|---|---|
| エンタープライズ / 大企業 共同 wiki | マルチユーザー / コラボ機能を作らない |
| 開発者専用 (cortex 系) | 一般人向けと矛盾 |
| Heavy Obsidian user | 既存ツールで満たされている、本アプリは別カテゴリ |
| データを cloud で共有したい人 | on-device only、cloud sync なし |
| Anki-like ガチ暗記学習者 | テスト / 不正解 UI を作らない |
