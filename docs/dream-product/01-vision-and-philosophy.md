# 01 — Vision & Philosophy (v2)

## 一文ビジョン (canonical)

> **「日常で触れたあらゆる情報をアプリに共有して、その情報は AI が読み解き・繋ぎ・要約しながら、今まで読んで保存した記事や bookmark で後で読む記事など、活用されないままの眠っている知識を自動で蓄積でき、自分唯一のものとして所用し、それを iPhone の Apple Intelligence をさらに進化させ差別化できる『新たな AI』として あなたに使えるようになる。必要に応じて、knowledge も現在使っている Gemini / ChatGPT などに共有して、さらに進化できる AI になる。そして、自分の秘書のように大事な情報を整理して要点を提示し、スピーディーに情報キャッチアップでき、必要に応じて自分の理解として腹落ちするまで会話で深堀りできるアプリ。」**

### 短縮 1 文 (内部議論用)

> 「日常で触れた情報を AI が体系化し、秘書のように要点を、家庭教師のように深堀りを、完全 on-device で提供する『あなた専用に進化した Apple Intelligence』」

---

## 中核哲学

すべての設計判断は、Karpathy のこの一文に集約される:

> "**You can outsource your thinking, but you cannot outsource your understanding.**"
> (思考は外部化できても、理解は外部化できない。)

### 一般人向け翻訳: 「秘書 + 家庭教師」が 1 つになった AI

| ロール | 担当する状態 | アプリでの実装 |
|---|---|---|
| **秘書** (Outsource Thinking) | 「あれ何だっけ」「これまとめて」 = 都度の問い | **News Clip 風カード** で要点提示 + **Chat** で質問に直接答える |
| **家庭教師** (Understanding) | 「これってどういう意味?」「もっと知りたい」 = 持続する理解 | **Card** で「今のあなたへ」surface + **Chat** で腹落ちするまで深堀り会話 |

→ **このアプリは「思考は委ねていい、ただし理解は委ねさせない」設計を貫く**。秘書として情報を捌き、家庭教師として理解を伴走する。

### なぜこの 2 ロールが大事か

- **「秘書だけ」アプリ** (ChatGPT / NotebookLM 等) は便利だが、ユーザーは「読んだ気・分かった気になって終わる」
- **「家庭教師だけ」アプリ** (Anki / 学習アプリ) は能動性を要求しすぎて続かない、生活との接点が薄い
- **両方を 1 つの体験で繋ぐ** = 「貯める → 聞く → 深堀って腹落ち」が無理なく回る = **既存に無いカテゴリ**

## Apple Intelligence を進化させる「あなた専用 AI」という位置づけ

### Base = Apple Intelligence (Foundation Models)

- iPhone に標準搭載される Apple Intelligence をそのまま base に
- 追加 API 課金ゼロ、SDK ハードルゼロ、デバイス変更時の引き継ぎ Apple 標準

### 差別化 = ユーザー固有の context で進化

- 一般的な ChatGPT / Gemini は「世界の知識」しか持たない
- 本アプリは「**あなたが日常で触れた情報の context**」を AI に注入し続ける
- 結果として、**「あなただけの過去・興味・読んだもの」を理解する AI** になる
- これが Apple Intelligence の生の状態には無い差別化

### 売り (ユーザー視点)

> 「Apple Intelligence は便利だけど、あなた個人の context を知らない。本アプリを使うと、Apple Intelligence が **あなた専用** に進化していき、あなたが読んだ・触れた・気になったすべてを覚えていて、聞けば答え、必要なら一緒に学ぶ AI になる」

## 「眠っている知識」の活性化が中核価値

ユーザーの実体験:
- ブラウザに 100+ タブを開いたまま「あとで読む」状態
- Pocket / Reading List / Bookmark が貯まる一方、読み返さない
- LINE / Slack で共有された記事を後で見ようと思って忘れる
- ChatGPT / Gemini との会話で得た insight が history に埋もれる

→ これらすべて **「活用されないままの眠っている知識」**。

本アプリの第一の責任は、**この眠っている知識を活性化すること**:
1. **共有を 1 タップに**: iPhone Share Sheet 経由で何でも投入
2. **読まなくても要点を**: AI が読み解いて要約・体系化
3. **必要な時に surface**: 過去の関連知識が会話の中で自動で呼び出される
4. **腹落ちまで伴走**: 興味あるものは深堀りで自分のものに

## 既存ツールでは何が足りないのか

| 既存ツール | できること | 足りないこと |
|---|---|---|
| ChatGPT / Claude / Gemini (chat 単体) | 何でも聞ける汎用知能 | 「あなたが読んだもの」が context にならない、毎回ゼロから |
| NotebookLM | 自分のソースに RAG | 蓄積が compound しない、ソースの再発見しかない |
| Pocket / Instapaper / Apple Reading List | 記事を貯める | 貯めるだけ、繋がりも要約も自動化されない、結局読まない |
| Apple Notes / Bear / Notion | 自由にメモ | 整理が人間任せ、続かない |
| Obsidian + LLM 統合 | 強力な wiki + AI | CLI / Markdown literacy 前提、デスクトップ必須、一般人ハードル高 |
| Anki / 学習アプリ | 能動的に覚える | 「貯める」が別アプリ、生活との接点なし |
| iOS Apple Intelligence (base) | 標準搭載・無料 | あなた固有の context を持たない、ジェネリック |

**結論**: 「読んだあらゆる情報を貯めて、AI が体系化し、必要な時に質問でき、理解したいときには深堀りもでき、Apple Intelligence をあなた専用に進化させ、必要なら他 AI に export も可能、すべて完全 on-device で動く、一般人向け iPhone アプリ」 は存在しない。

これが本アプリの位置。

## 設計原則 (11 個)

### 普遍原則 7 つ (外部研究: Karpathy / SAGE / Tableau AKG / cortex 由来)

#### 1. Knowledge compounds (複利的に蓄積する)

- 新しい情報を入れるたびに、既存の知識が更新される
- RAG の「クエリ時に都度検索」ではなく、**永続的に成長する成果物**
- 「使うほど賢くなる」を本物にする

#### 2. 「説明文 (business context)」が検索精度の本体

- 単純な名前 / キーワードだけでは意味検索が効かない
- 各ノードに固有の「何を意味するか / なぜここにあるか」が必須
- AI が書く要約 (= ノード固有の説明) が embedding の入力になる

#### 3. bookkeeping は LLM が全部やる

- 人間: ソースを curate、問いを立てる、判断する
- LLM: 要約・繋ぐ・矛盾検出・一貫性維持・健全性チェック
- **「人間が wiki を諦めるのは維持負荷が価値より速く増えるから」** を反転させる

#### 4. Runbook pattern (次のアクションを内蔵)

- AI の答え / カードに **「次に何をすべきか」** を常に同梱
- ユーザーは「次どうする?」で迷わない

#### 5. 自己進化 (使うほど賢くなる)

- 情報投入 → 概念ページ更新 → 質問 → 結果ファイリング → 次の問いが surface
- フィードバックループが切れない設計

#### 6. ハルシネーション位置を意識的に設計する

- LLM が介在する場所は **抽出時のみ** に限定
- 検索 / 表示 / 編集は決定論的 (LLM 介在ゼロ)
- 「答え自体がハルシネーション」ではなく「材料がレビュー済の事実」

#### 7. AI が書く前提だから成立する設計

- 「人が毎回 5 種のメタデータを書け」は維持不能
- AI が書く時代だから「説明文をモノの場所に固定」「健全性 lint 自動」が機能する
- 人間編集 UI を最小化、AI が書く + 人間が approve/訂正

### 本アプリ固有原則 4 つ

#### A. 完全 on-device (プライバシー絶対)

- すべての処理が iPhone 内で完結
- クラウド API 一切なし (ChatGPT / Claude / Gemini API への自動送信ゼロ)
- 翻訳・要約・抽出・検索すべて Foundation Models (Apple Intelligence)
- **唯一の例外**: ユーザーが意図的に export して別 app に渡すのは OK (= ユーザー主体)

#### B. Calm UX (不安喚起しない)

- 通知ゼロ・バッジゼロ・「未読」赤丸ゼロ・「streak」連続日数ゼロ
- 開けば変化が見える、開かなくても罪悪感ゼロ
- 学習アプリにありがちな「サボると怒られる」感を一切作らない

#### C. 受動的 + 能動的の両モード共存

- **受動モード**: 通勤・寝る前に 1 タップで「今のあなたに」surface される
- **能動モード**: 「もっと知りたい」と思ったら会話で深堀りできる
- どちらも強要しない、ユーザーが選ぶ

#### D. 一般 iPhone ユーザーが今日から使える

- CLI / Markdown / Obsidian / プログラミング知識 一切不要
- iPhone Share Sheet を知っていれば使い始められる
- 設定不要、デフォルトで価値が出る

## 「やる / やらない」の境界

### やる

- 日常で触れたあらゆる情報の摩擦ゼロ取り込み (Web / PDF / 写真 / AI 会話)
- AI が読み解き・繋ぎ・要約 (秘書)
- 質問に直接答える Chat (秘書)
- News Clip 風カードで要点提示 (秘書)
- 腹落ちまでの深堀り会話 (家庭教師)
- 興味カードで能動学習も可能 (家庭教師)
- 完全 on-device 動作 (Apple Intelligence)
- **すべての knowledge を zip / markdown で export 可能** (user が選んだ手段で外部に持ち出せる)
- export 結果を user が **手動で** Gemini / ChatGPT などに渡す ことが可能
- 一般 iPhone ユーザー (技術知識ゼロでも) が使える

### やらない (詳細は 08-non-goals.md)

- 「毎日開いて連続記録を維持しよう」ゲーミフィケーション
- ユーザーに「整理して」を要求する UI
- **アプリ内から自動でクラウド LLM API を呼ぶ** (Gemini/ChatGPT/Claude API への自動送信)
- 「正解 / 不正解」を強いるテスト UI
- マルチユーザー / 共有 / コラボレーション
- 課金 / サブスク / 広告
- web search を自動で発火 (V2 で BYOK opt-in 検討候補)

## 成功の定義 (5 項目)

「このアプリが成功した」と言える状態:

| 条件 | 中身 |
|---|---|
| **眠っている知識の活性化** | ブラウザのタブ・Pocket・Bookmark が活きてくる、「読まずに忘れていた」が「読まなくても要点が頭に入る」に変わる |
| **「あなた専用 AI」感** | 「他の人の Apple Intelligence と、自分の Apple Intelligence が違う」と感じる、自分の context を持つ AI として育つ |
| **維持コスト ≈ 0** | ユーザーが「保存するだけ」で知識が育つ感覚を持続できる、「整理しなきゃ」ストレスが消える |
| **理解の増幅** | 「読んだものを自分のものにする」体験ができる、深堀り会話で腹落ちする |
| **既存に無いカテゴリの製品** | 「AI が wiki を作って育てて教えてくれる、一般人向けの全部入りモバイル知識ベース」として認知される |

## このビジョンが正しいかの確認方法

「これは本物か」を判断する短い問い:

1. 「3 ヶ月使ったら、自分の頭の中の地図が広がった感覚があるか?」
2. 「保存したものを、後で『あれ何だっけ』と思った時に、ちゃんと出てくるか?」
3. 「『分からない』が『分かった』に変わる瞬間が、アプリの中で起きるか?」
4. 「ブラウザに開きっぱなしのタブが減ったか?」
5. 「他のアプリでは得られない、この体験が忘れられない、と思えるか?」

→ Yes が 4/5 以上なら、ビジョンは正しい方向。
