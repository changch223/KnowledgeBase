# リリース前 実機スモークテスト（1本化チェックリスト）

各 spec に散っていた「残: 実機検証」を、**出荷前に実機で1回通せばよい**形に統合したもの。
上から順に。**P0 = 通らなければ出荷不可 / P1 = 強く推奨 / P2 = できれば**。

> 準備: 実機（Apple Intelligence 対応機）に **クリーンインストール**。
> 検証は **記事を20〜30件保存して"育った状態"** を作ってから行う。

---

## P0 — コア体験（落ちたら出荷不可）

- [ ] **保存（URL）**: Safari で記事を共有 → Knowledge Base に保存できる
- [ ] **保存（＋ボタン URL）**: URL を入力して保存できる
- [ ] **保存（メモ）**: テキストを貼り付けて保存できる
- [ ] **保存（ファイル）**: PDF / テキストファイルを選択して保存できる
- [ ] **保存（写真 OCR）**: 文字の写った画像を保存 → テキスト化される
- [ ] **保存（音声）**: ＋ボタン「音声」で m4a 等を選択 → 文字起こしされ知識化される
- [ ] **音声共有**: Voice Memos / Files から m4a を共有 → アプリ起動で文字起こし → 概念生成
- [ ] **知識抽出**: 保存後しばらくで essence / 要点 / タグが付く（「整理中」が消える）
- [ ] **概念ページ生成**: 関連記事が束ねられた概念ページができる（人物/テーマなど）
- [ ] **iKnow フィード**: 概念まとめ + 新着記事 + 横スクロール棚が表示・スクロール滑らか
- [ ] **AIチャット**: 保存知識に質問 → **番号引用 `[1][2]` 付き**で回答、出典タップで記事へ
- [ ] **チャット該当なし**: 保存外の質問 → 「一般的な回答」バッジが出る（捏造しない）
- [ ] **検索**: ライブラリでキーワード検索 → 関連度順で出る
- [ ] **クラッシュなし**: 上記一連でアプリが落ちない

## P0 — 要確認機能（spec 092-095 で追加・修正）

- [ ] **本文訂正**: 記事詳細のツールバー「訂正」ボタン → 指示を入力（例: `Claude Code を gloadcod と誤認識しています`）→「訂正する」→ シートを閉じても上部バナーで処理継続 → 完了後に本文＋概念が更新される
- [ ] **用語補正**: 音声に「Claude Code」「Anthropic」等の専門語が含まれる → 文字起こし後に正しい表記になる（完全一致でなくてもある程度改善）
- [ ] **英語記事翻訳**: 英語の記事を保存 → 要点・概念名が日本語で生成される
- [ ] **見直し完了通知**: 訂正完了後、どの画面にいてもバナーが出て「確認」シートを開けること

## P1 — 整理・編集

- [ ] **分類の確認**: 設定 →「分類の確認」で要確認タグに正しい分野を選べる（学習に記録される）
- [ ] **タグ・分野管理**: 改名 / 統合 / 非表示ができ、フィードに反映される
- [ ] **「今すぐ整理」**: ボタンで整理が走り、対象件数が表示される（途中中断→次回続きから）
- [ ] **概念ページ編集**: 本文編集（保存後 bodyEditedByUser=true で再生成スキップ）/ 非表示 / ピン
- [ ] **削除**: 記事をスワイプ（List）/ 長押し（概念詳細・フィード）で削除できる

## P1 — 同期・多言語

- [ ] **iCloud 同期（任意）**: 設定で ON → 別端末 or 再インストールでデータ復元
- [ ] **多言語音声**: 中国語の音声を共有 or ＋ → 中国語で転写後、日本語に翻訳して概念化される
- [ ] **多言語 URL**: 中国語・韓国語のページを保存 → 日本語の要点になる

## P2 — 仕上げ・周辺

- [ ] **オンボーディング**: 初回起動の4ページが現行内容（Page4 = AIチャット導線）
- [ ] **ダーク/ライト**: 切替で表示が崩れない
- [ ] **ウィジェット**: ホーム画面に追加できる
- [ ] **Safari 拡張**: 設定の手順で有効化 → 閲覧中のページを保存できる
- [ ] **空状態**: データ0件で各タブが優しい空表示（クラッシュしない）

## ログで確認できること（目安）

正常動作時のログパターン（Xcode コンソール）:

```
category learning container: local-only          ← 学習ストア正常
classify 'xxx' -> 'テクノロジー' [High]           ← 分類が動いている
knowledge chunked result: status=succeeded        ← 知識抽出成功
concept synthesis succeeded for ...               ← 概念合成成功
translate prep: detected=japanese ... override=true ← 日本語は翻訳スキップ (正常)
translate prep: detected=english ...              ← 英語記事を日本語化中
```

## 既知の無害ノイズ（無視してよいログ）

- `updateTaskRequest failed ... Code=3` — CloudKit BGTask ノイズ、無害
- `nw_protocol... udp` / `tcp_input ... CLOSED` — ネットワーク系、無害
- `CoreData: debug: WAL checkpoint` — DB 内部処理、無害
- `Failed to set up CloudKit integration for store: CategoryLearning.store` — **旧ファイルの残骸**。新規インストールでは出ない。クリーンインストール後は消える
- `Sandbox restriction` / `SandboxExtension` 系 — 無害

---

## 不具合が出たら

- Xcode コンソールを採取 → どの操作で・どのログが出たかをメモして連絡
- クラッシュ（EXC_BAD_ACCESS / EXC_BREAKPOINT が Resume 不可）は P0 として最優先
- 機能が動かない場合は操作手順（何を保存・何を質問）を添えると原因を絞りやすい
