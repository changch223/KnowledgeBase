# リリース前 実機スモークテスト（1本化チェックリスト）

各 spec に散っていた「残: 実機検証」を、**出荷前に実機で1回通せばよい**形に統合したもの。
上から順に。**P0 = 通らなければ出荷不可 / P1 = 強く推奨 / P2 = できれば**。

> 準備: 実機（Apple Intelligence 対応機）にインストール。検証は **記事を20〜30件保存して“育った状態”** を作ってから。
> 1台はクリーンインストール（前バージョンの壊れた CategoryLearning.store を消すため）。

---

## P0 — コア体験（落ちたら出荷不可）

- [ ] **保存（URL）**: Safari で記事を共有 → Knowledge Base に保存できる
- [ ] **保存（＋ボタン）**: URL / メモ / ファイル(PDF) / 写真 / 音声 の各モードで保存できる
- [ ] **写真OCR**: 文字の写った画像を保存 → テキスト化される
- [ ] **音声**: m4a を共有 or ＋から音声 → 文字起こしされ知識化される
- [ ] **知識抽出**: 保存後しばらくで essence / 要点 / タグが付く（「整理中」が消える）
- [ ] **概念ページ生成**: 関連記事が束ねられた概念ページができる（人物/分野など）
- [ ] **iKnow フィード**: 概念の超まとめ + 新着 + おすすめが表示・スクロール滑らか
- [ ] **AIチャット**: 保存知識に質問 → **番号引用 `[1][2]` 付き**で回答、出典タップで記事へ
- [ ] **チャット該当なし**: 保存外の質問 → 「一般的な回答」バッジが出る（捏造しない）
- [ ] **検索**: ライブラリでキーワード検索 → 関連度順で出る
- [ ] **クラッシュなし**: 上記一連でアプリが落ちない

## P0 — 今セッションの修正点（必ず確認）

- [ ] **学習ストア local-only**: 起動ログに `category learning container: local-only`（`Store Removed` が出ない）
- [ ] **翻訳安定化**: 英語の長い記事を保存 → ログに `translate prep: ... override=true`、`translationd crashed` の連発が**出ない**、取り込みが極端に遅くない
- [ ] **分類 confidence**: ログに `classify '...' -> '...' [High/Medium/Low]`、人名など曖昧語が「その他」に寄る
- [ ] **overflow なし**: ログに `4091`/`exceededContextWindowSize` が出ない（概念合成が essence-list に劣化しない）
- [ ] **見直し完了ポップアップ**: 本文の見直し/訂正 → 完了通知がどの画面でも出る

## P1 — 整理・学習・編集

- [ ] **分類の確認**: 設定 →「分類の確認」で要確認タグに正しい分野を選べる
- [ ] **学習が効く**: 分野を直した後、似たタグの分類が改善（few-shot）
- [ ] **タグ・分野管理**: 改名 / 統合 / 非表示ができ、フィードに反映
- [ ] **本文の見直し**: 知識ベース基準で誤記（gloadcode→Claude Code 等）が直る
- [ ] **生成のカスタマイズ**: 抽出の方向性指定が効く
- [ ] **「今すぐ整理」**: ボタンで整理が走り、件数表示が出る（途中中断→次回続き）
- [ ] **概念ページ編集**: 本文編集・非表示・ピンができる
- [ ] **削除**: 記事をスワイプ/長押しで削除できる

## P1 — 同期・多言語

- [ ] **iCloud 同期（任意）**: ON → 別端末 or 再インストールでデータ復元
- [ ] **多言語**: 中国語の音声/URL → 日本語で概念化される
- [ ] **英語記事**: 日本語の要点になっている

## P2 — 仕上げ・周辺

- [ ] **オンボーディング**: 初回起動の4ページが現行内容（Page4 = AIチャット）
- [ ] **ダーク/ライト**: 切替で表示が崩れない
- [ ] **ウィジェット**: ホーム画面で表示される
- [ ] **Safari 拡張**: 設定どおり閲覧ページを保存できる（使う場合）
- [ ] **空状態**: データ0件で各タブが優しい空表示

## 不具合が出たら
- ログ（Xcode コンソール）を採取 → どの操作で・どのログが出たかをメモ
- 既知の無害ノイズ: `updateTaskRequest failed ... Code=3`（メイン store CloudKit）、`nw_protocol... udp`、`Sandbox restriction` 系は無視可
- クラッシュは P0 として最優先で記録
