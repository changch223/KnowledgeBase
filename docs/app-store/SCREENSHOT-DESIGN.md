## 0. 確定した設計方針

| 決定事項 | 内容 |
|---|---|
| **ビジュアル枠** | 上帯キャプション + 端末フレーム (和紙背景) — プレミアム・「設計された」印象 |
| **枚数** | **5 枚**（要点を外さず制作も現実的） |
| **1 枚目ヒーロー** | 「**読んだことが、勝手にまとまる。**」（自動体系化 = VISION の核心） |
| **ブランド** | 和紙 + 墨絵、明朝見出し、青海波モチーフ、余白 (間) |
| **端末の見せ方** | **実機ベゼル込みの端末フレーム**（角丸スクショのみより立体感・信頼感が出る、プレミアム枠と整合） |
| **#4 取り込み画面** | **＋ボタンの 5 モード segmented**（URL/メモ/ファイル/写真/音声が一目で分かる、共有シート案は不採用） |
| **撮影モード** | Light（和紙背景が最も映える）、9:41、フル電波・充電 |

### v2 ブラッシュアップ（2026-07-04 追加確定、実装 = `ScreenShot/appstore/build.py`）
| 決定 | 内容 | 根拠（ベストプラクティス） |
|---|---|---|
| 端末下端ブリード | 端末 +18% 大型化・下端カット | Apple 公式の定番。UI を大きく、死に余白を消す |
| 藍アクセント | 見出しの核心語 1 語だけ `#3A4A63`（勝手に/百科事典/根拠付き/自動/知識） | サムネイル一覧で埋もれない視線フック（朱書き調） |
| コールアウトチップ | 1 枚 1 つだけ、見どころを指す和紙ピル + 藍ドット（要点を先出し/31記事を1つに/出典付き回答/自動で分類/要約も自動） | 「1 枚 1 メッセージ」を 0.5 秒で伝える |
| その他 | 見出しベースライン 5 枚固定・薄ベゼル 12px・柔らかい大影・青海波は下コーナーに残存 | シリーズの一貫性・上質感 |

**物語の弧（5 枚を貫く 1 本のストーリー）**  
まとまる → 事典ができる → 質問できる → 何からでも入る → 全部端末内で安全

## 1. マスターレイアウト（全 5 枚共通の下地）

サイズ: **1320 × 2868 px（iPhone 6.9″ / Pro Max、縦）**

```
┌──────────────────────────────────┐  ← 0px
│                                          │
│   ［見出し・明朝太字・墨色］               │  上帯 = 上部 22%（≈630px）
│   ［サブ・ゴシック細字・墨中間色］          │  背景 = 和紙 #F4EFE6
│                                          │
│ ──────────────────────────────── │  ← 墨のヘアライン 1.5px（#D9D2C4 系）
│                                          │
│        ╭────────────────────╮          │
│        │                      │          │
│        │   実機スクリーンショット   │          │  端末フレーム
│        │   （端末ベゼル込み）      │          │  幅 ≈ 画面の 84〜88%
│        │                      │          │  中央寄せ、微かなドロップシャドウ
│        │                      │          │  背景 = 和紙 #F4EFE6
│        ╰────────────────────╯          │
│                                          │
│  ∿∿∿∿ 青海波の薄い連続文様 ∿∿∿∿          │  下部 ≈8%、藍 #3A4A63 を低透明度
└──────────────────────────────────┘  ← 2868px
```

### タイポグラフィ・色（厳密指定）

| 要素 | 指定 |
|---|---|
| 見出し | 明朝体（serif）**太字**、墨色 `#1C1B19`（`sumiInk`）、**72〜84px**、字間やや詰め、最大 2 行 |
| サブ | ゴシック（sans）細〜中字、`#4A4640`（`sumiMid` 相当）、**36〜42px**、1 行 |
| ヘアライン | `#D9D2C4`（`sumiRule` 相当）、**1.5px**、帯の下端に 1 本 |
| 帯背景 | 和紙 `#F4EFE6`（`washiBackground`）※端末背景と同色で一体化 |
| 端末影 | 墨 8% 程度の柔らかい影、ぼかし大（安っぽい黒影にしない） |
| 青海波 | 藍 `#3A4A63`、透明度 8〜12%、下端のみ、主張しすぎない |

> **原則**: 帯と端末背景は**同じ和紙色**にして「1 枚の和紙の上に端末が置かれている」ように見せる。境界はヘアライン 1 本だけ。装飾は青海波の薄い文様のみ。過剰にしない = 間 (ma) を活かす。

## 2. 5 枚の確定構成

| # | 画面 | 見出し（明朝・帯） | サブ（ゴシック・小） | この枚で伝えること |
|---|---|---|---|---|
| **1** | **ナレッジフィード**（概念カード + 要点先出し） | **読んだことが、勝手にまとまる。** | AIがテーマごとに要点を先に見せる | 保存するだけで自動体系化される「魔法」 |
| **2** | **概念ページ（Wiki）詳細** | **AIが、あなただけの百科事典を編さん。** | 複数の記事を1ページに束ね、関連へ自動リンク | 成果物 = 自分専用の事典が育つ |
| **3** | **AI チャット**（番号引用付き回答） | **あなたの知識に、根拠付きで答える。** | 引用番号をタップで元記事へ | 貯めた知識をそのまま使える |
| **4** | **取り込み**（共有シート or ＋の5モード） | **URL・写真・PDF・音声。どこからでも。** | 英語・中国語も日本語に自動翻訳 | 入口が広い（翻訳もここで訴求） |
| **5** | **プライバシー**（設定 or オンボーディング） | **すべて端末内。データは外に出ない。** | 広告なし・トラッキングなし | 信頼の締め（クラウド依存競合との差） |

> **最初の 3 枚で価値が完結**するよう #1〜#3 に主要価値を集中。#4/#5 は幅と信頼の補強。

## 3. 各画面の撮り方（実機キャプチャ指示）

### 撮影前の共通準備
- **デモデータを完走**: 実記事を 15〜20 本保存し、AI 整理（概念生成・カテゴリ・要点・チャット引用）を**完了させてから**撮る。空カードや「整理中…」は写さない。
- 固有名詞が不都合なら中立なダミー記事に差し替え（例: 一般的な技術・経済ニュース）。
- Light モード / 9:41 / フル電波・充電。

### #1 ナレッジフィード（ヒーロー）
- タブ: **Knowledge**（`newspaper.fill`）。
- 概念カードが **3〜4 枚**縦に見える位置までスクロール。各カードの**要点の箇条書き（青い•）が読める**状態。
- 上部の「新着」横棚が入ると情報が散るので、**概念カードが主役**になる位置で撮る。
- 撮った実機画像を端末フレームに合成 → 上帯に見出し。

### #2 概念ページ（Wiki）詳細
- 概念カードをタップした詳細画面。**大見出し（明朝の概念名）+ 要点 + 子トピックチップ + 記事数**が 1 画面に入る位置。
- 「関連ページ」への青いインラインリンクが 1 つ見えると「自動リンク」が伝わる。
- 種別バッジ（人物/概念/プロジェクト）も見えると良い。

### #3 AI チャット（引用付き）
- ユーザーの質問（右・青）+ AI 回答（左）で、**回答本文に番号引用 `[1]`** + 下部に「出典」番号リストが見える状態。
- 質問例は具体的に（例: 「最近保存したAI関連の要点は?」）。回答に引用が 2 件以上付くように事前に良い質問を選ぶ。

### #4 取り込み（入口の広さ）★確定: ＋の 5 モード
- ＋ボタンの追加シートで **5 モード（URL / メモ / ファイル / 写真 / 音声）** の segmented が見える画面。
- サブで翻訳を訴求するので、入口の種類が一目で分かるこの画面にする（共有シート案は不採用）。

### #5 プライバシー
- **案A**: オンボーディングの「すべて端末内」を説明するページ。
- **案B**: 設定画面で iCloud（自分のプライベート DB のみ）+ トラッキングなしが分かる箇所。
- 実機に文言が薄ければ、帯の見出し + サブで主張を補う（UI は静かな背景でよい）。

## 4. 制作ワークフロー（Figma / Sketch、最速手順）

1. **アートボード 1 枚**を 1320×2868 で作成し、§1 のマスターレイアウトを組む（帯・ヘアライン・青海波・端末フレームのプレースホルダ）。
2. これを**コンポーネント化**し、5 枚に複製。
3. 各複製に §3 の実機スクショを流し込み、§2 の見出し・サブを差し替え。
4. 見出しの明朝は **Noto Serif JP / 游明朝 / ヒラギノ明朝** のいずれか。太さは Bold。
5. 6.9″（1320×2868）を主とし、必要なら 6.5″（1284×2778）にリサイズ書き出し。
6. 書き出し: PNG、sRGB、アルファなし。

## 5. 生成 AI / デザイナー向け prompt

### 5-0. アートディレクション（全 prompt に前置き）
```
Art direction: refined Japanese "washi + sumi-e" aesthetic, ukiyo-e restraint.
Palette: warm off-white washi paper (#F4EFE6), sumi ink black (#1C1B19),
soft indigo accent (#3A4A63), muted stone grey. No neon, no heavy gradients.
Elegant Mincho (serif) headings. Motifs: seigaiha (青海波) waves, thin hairline
rules, generous negative space (ma / 間). Calm, premium, quiet, literary.
NOT flashy, NOT corporate-tech-blue, NOT cluttered.
```

### 5-1. 下地テンプレート（端末・UI・文字なしの背景 1 枚）
```
[Art direction above]
A minimal App Store screenshot BACKGROUND template, portrait 1320x2868.
Top 22%: a solid warm washi band reserved for a Japanese Mincho headline (leave empty).
A single 1.5px sumi hairline separates the band from the area below.
Below: the same washi paper texture as a clean stage for a centered phone mockup.
A faint seigaiha wave motif along the very bottom edge, ~10% opacity indigo.
Nothing else: no device, no UI, no text. --ar 1320:2868 --style raw
```
> 実運用: この下地に §3 の実機スクショを端末フレーム込みで中央合成し、上帯に §2 の見出し（明朝太字）+ サブ（ゴシック）を載せる。

### 5-2. 各スライドのキャプション流し込み（コピペ用）

```
Slide 1 — 見出し: 読んだことが、勝手にまとまる。 / サブ: AIがテーマごとに要点を先に見せる
Slide 2 — 見出し: AIが、あなただけの百科事典を編さん。 / サブ: 複数の記事を1ページに束ね、関連へ自動リンク
Slide 3 — 見出し: あなたの知識に、根拠付きで答える。 / サブ: 引用番号をタップで元記事へ
Slide 4 — 見出し: URL・写真・PDF・音声。どこからでも。 / サブ: 英語・中国語も日本語に自動翻訳
Slide 5 — 見出し: すべて端末内。データは外に出ない。 / サブ: 広告なし・トラッキングなし
```

### 5-3. 禁止事項（negative prompt）
```
No neon, no glossy 3D, no corporate tech-blue gradients, no stock-photo people,
no cluttered UI collage, no fake metrics/charts, no drop-shadow-heavy cards,
no emoji, no English marketing buzzwords in the headline band.
```
